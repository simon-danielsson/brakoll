=== Adding a new issue ===

When you want to add something to your issue list you simply type it out in your project
(I would advise you to create a snippet for this; for example "issue").

Example:
*brakoll - d: fix formatting issue in debug statement, p: 10, t: debug, s: open

d: description of the issue (obligatory)
p: priority from 0 to infinity where the highest number is the most critical priority (optional - fallback: 0)
t: tag (optional - fallback: n/a)
s: status [ (op)en | (pr)ogress | (cl)osed ] (optional - fallback: open)

Important:
* Issues are currently only single-line! If you want a long description,
  write it all on a single line or alternatively refer to a bigger document
  somewhere else inside your description.
* The prefix "*brakoll" is required but whatever is before it on the line
  is of no importance to the parser and will be ignored,
  e.g "#", "//", "--" or any other comment syntax.

=== Flags & Subcommands ===

All the issues listed, sorted by priority and status:
$ brakoll

Filter issues by tag:
$ brakoll -s <tag>

Filter issues by status:
$ brakoll -s <status>

Summary of all issues:
$ brakoll summary

Display help and version information:
$ brakoll help
